Restore-VM-Without-Snapshot

  • Go to devops-docs/Emergency-Scenario/Proxmox

  • Choose Option b.) and SSH into the host, where the VM is on and follow the instructions

  • We basically Pipe the output of the borg extract command into th qmrestore command
     which then takes a few minutes (20min +, depending on VM-Size)

Restore-Artifactory-VM-Without-Snapshot

  • After the restore explained above, the new VM will probably miss one Harddisk

  • We need to identify the disk, that went missing during the restore, by checking the disks on
    the old-broken VM and compare them to the ones from the restored VM, one should be missing

  • Once identified, we search for it at:
            
            /var/lib/VZ/images/${VM-ID}/${DISKNAME}
       

  • Now we simply copy this disk Image-File, to the same location on the restores VM
    ans then give it a name, that matches the restored VMs VM-ID

  • After that we als need to adjust the config at:
            
            /etc/pve/qemu-server/${VM-ID}
       

    In here we just need to add the new Disk likt the ones already inserted into the config
    but of course with the VM-ID of the newly restored VM.
