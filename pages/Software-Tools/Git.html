General-Git-Tips

Adjust Commit History


  • interactive rebase starten ( for ex. 3 commits should be adjusted ~3 ): git rebase -i Head~3

  • Commit lines can then be cut, moved, pasted as desired, directly in opened vim-editor

Squash


  • To squash in Git means to combine multiple commits into one.

  • interactive rebase starten ( for ex. 3 commits should be adjusted ~3 ): git rebase -i Head~3

  • Search for commit(s) that need to be squashed

  • Replace the 'pick' in front of the commit line with a 's' and close the editor

  • In the next vim window, we see what commits get taken into the squash and we can
    determine a new commit-message or just comment out all, except the one we want to keep.

Rebase

Interactive Rebase Prefixes

     
  • p – pick: Use the commit as is                                                                                            r – reword: Edit the commit message
    •  
  • e – edit: Pause to amend the commit                                                                                  s – squash: Combine with the previous commit and keep both messages
    •  
  • f – fixup: Combine with the previous commit and discard this one's message                  x – exec: Run a shell command
    •  
  • b – break: Stop here during rebase (manually continue later)                                            d – drop: Remove the commit
    •  
  • l – label: Label the current commit for later use                                                                 t – reset: Reset HEAD to a labeled commit
    •  
  • m – merge: Create a merge commit using labeled commits

gitignore

How to Use .gitignore

     
  • Create a file named .gitignore in the root of your repository if it doesn't exist.
    •  
  • Add patterns to specify files, file types, or directories to ignore.

Ignore Specific Files

     
  • filename.txt → ignores a specific file in the root directory.
    •  
  • path/to/file.log → ignores a specific file in a subdirectory.

Ignore File Extensions

     
  • *.log → ignores all files ending in .log.
    •  
  • *.tmp → ignores all temporary files.

Ignore Folders

     
  • foldername/ → ignores the folder and all its contents.
    •  
  • logs/ → ignores the logs directory.

Negating Patterns (Include After Ignore)

     
  • *.log ignores all .log files.
    •  
  • !important.log keeps important.log tracked even if above pattern exists.

Important Notes

     
  • .gitignore only affects files not yet tracked by Git.
    •  
  • To stop tracking a file already in Git: git rm --cached ${FILENAME}
    •  
  • Use git status to confirm ignored files are not staged.
