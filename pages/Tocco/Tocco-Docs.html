  • Ensure your docs directory contains a conf.py file for Sphinx configuration and an index.rst file as the root of your documentation.
  • If you encounter any missing dependencies, you can add them to the requirements.txt file and install them using pip install -r requirements.txt

    • Prerequisites

    Ensure you have the following installed on your local machine:


    • Python 3

    • Sphinx

    • Sphinx Read the Docs theme

    • Graphviz (if your documentation uses it)


    These can b installed with:

          
            
         apt install -y --no-install-recommends graphviz python3-sphinx python3-sphinx-rtd-theme
       
   

    Step-by-Step-Guide

    Clone Your GitLab Repository:

        
            
         git clone ${your-repo-url}
         cd ${your-repo-directory}

 

    Create a Virtual Environment: (Not necessary)

        
            
         python3 -m venv venv
         source venv/bin/activate

 

    Install Dependencies:

    Install the necessary dependencies listed in your .gitlab-ci.yml file.

        
            
         apt install -y --no-install-recommends graphviz python3-sphinx python3-sphinx-rtd-theme

 

    Make Your Changes:

    Edit or create new documentation entries in the docs directory of your repository.

    Build the Documentation:

    Use Sphinx to build your documentation locally.

        
            
         sphinx-build -b html docs _build

 

    View the Documentation:

    You can serve the built documentation locally using Python's built-in HTTP server.

        
            
         cd _build
         python3 -m http.server 8000

 

    Access the Documentation:

    Open a web browser and go to http://localhost:8000 to view your built documentation.
