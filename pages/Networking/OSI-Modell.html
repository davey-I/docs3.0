Application-Layer

  • Used by Network Applications, that means applications that use the Internet.
    for ex. Browser Applications (Chrome, Firefox etc.), Outlook, Slack etc.

  • Application Layer includes and provides Protocols, that are used for Network application services:
       
    • HTTP/HTTPS-Protocol | All in Browser
      •  
    • FTP: File Transfer Protocol
      •  
    • SMTP: Mailstuff
      •  
    • Telnet: Virtual Terminals

    • Presentation-Layer

  • Receives Data from Application Layer, formatted as strings and numbers

  • Presentation Layer converts these to machine understandable binary format
    for ex. conversion from ASCII to EBCDIC format.

  • The Translation Process is then followed by a the compression process, that reduces the
    amount of data used. This can be lossy or lossless

  • The last process is the data encryption using SSL

    • Session-Layer

    Helps with setting up and managing connections using APIs

  • Takes care of the authentication process during connections build up

  • Takes care of the authentication process during connections build up

  • Helps in session management, for all sub-sessions opened during for ex. file access via Web

  • Everything up to here ( including Session-Layer) is done by Web-Browser

    • Transport-Layer

    Segmentation

       
    • Data received from Session layer are devided into small units called Segments

      •  
    • Each Segments contains a Source-Port , Destination-Port and Sequence Number

      •  
    • Port-Numbers help to direct each Segment to the correct Application

      •  
    • Sequence-Number helps to re-assemble Segments in the correct Order
