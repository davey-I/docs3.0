OSI-Modell

Application-Layer

  • Used by Network Applications, that means applications that use the Internet.
    for ex. Browser Applications (Chrome, Firefox etc.), Outlook, Slack etc.

  • Application Layer includes and provides Protocols, that are used for Network application services:
       
    • HTTP/HTTPS-Protocol | All in Browser
      •  
    • FTP: File Transfer Protocol
      •  
    • SMTP: Mailstuff
      •  
    • Telnet: Virtual Terminals

    • Presentation-Layer

  • Receives Data from Application Layer, formatted as strings and numbers

  • Presentation Layer converts these to machine understandable binary format
    for ex. conversion from ASCII to EBCDIC format.

  • The Translation Process is then followed by a the compression process, that reduces the
    amount of data used. This can be lossy or lossless

  • The last process is the data encryption using SSL

    • Session-Layer

    Helps with setting up and managing connections using APIs

  • Takes care of the authentication process during connections build up

  • Takes care of the authentication process during connections build up

  • Helps in session management, for all sub-sessions opened during for ex. file access via Web

  • Everything up to here ( including Session-Layer) is done by Web-Browser

    • Transport-Layer

    Segmentation

       
    • Data received from Session layer are devided into small units called Segments

      •  
    • Each Segments contains a Source-Port , Destination-Port and Sequence Number

      •  
    • Port-Numbers help to direct each Segment to the correct Application

      •  
    • Sequence-Number helps to re-assemble Segments in the correct Order

     

    Flow-Control

     
        
    • Controls the amount of data being transmitted

      •   
    • If for ex. a client is connected to a server and the server is capable of 100Mbps but
      the client can process data at max. 10Mbps

      •   
    • If the client downloads a file and the server starts to send data at 50Mbps
      the client can tell the server with the help of Transport-Layer to slowdown to
      max 10Mbps or the other way around if server starts too slow

    Error-Control

       
    • If some data does not arrive at the destination Transport Layer uses Automatic Repeat Request
      schema to re-transmit the lost or corrupted data.  
      • To each segment a Checksum gets added, to detect corrupted data

    TCP & UDP

  • Protocols used int Transpor-Layer are:  
         
    • TCP (Transmission Control Protocol) Connection Oriented Transmission
        provides a feedback if data transmission was successful or not and initializes retransmission if error
      •    
    • UDP (User Datagram Protocol) Connection less Transmission
        is faster because no feedback about successful data transmission is done.
      •  

    • Network-Layer

  • Data units in the Network-Layer are called packets

  • For ex. Routers reside in the Network-Layer

  • The functions of this Layer are:
       
    • Logical Adressing: IPV4 & IPV6 
      •  
    • Path determination
      •  
    • Routing
