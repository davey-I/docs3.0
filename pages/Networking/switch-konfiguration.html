how-to-switch-konfigurieren

 Config of a new Zyxel Switch after Tocco moved back to upper floors
and construction work in office was done Oct25.



- Plugin switch to power and Connect with local machine via Ethernet Cable
  Port should not matter to much for ex. Port 1

- Find out what the default Port of the switch is, via google or manual.
   also check what are the default Login credentials. This is often something like:

     - Port: 192.168.1.... 
     - admin / 1234  or admin / admin ...etc.

-  If we try to connect to the switch by entering the default Port into the browser,
   we will not yet be able to reach the switch settings. Because the switch has no DHCP
   and therefore our device is not given a valid IP-Adress. We can also see that if we execute
   ifconfig in the shell. Our machines won't have a IPv4 adress assigned.

   
Only thing imaginable, that could make this work auto.: If we
plugin the switch to a already existing Network, that has a DHCP-Server in it.
Maybe but rather unlikely too 


- So we need to adjust the network config of the local machine ( from where we configure switch )
  ourselves. On Ubuntu via Gui ( Settings - Network - PCI Ethernet "cog-icon" )
  also make sure the Switchbutton Icon is turned to "On" This mean we are connected to the switch phys..
 
  After selecting the "cog icon" choose IPv4. In these settings we need to switch from "Automatic (DHCP)" to "Manual"
  next we add a IPv4 Adress and a subnetmask. 

  
What IPv4 and what Subnetmask ?

    This is defined by the Switches Default Port and Subnetmask Values. So if for ex.
    the default Port is 192.168.1.1 and Subnetmask: 255.255.255.0 our
    Devices Network Interface needs to be given a IP within that range. For ex. 192.168.1.42 or whatever...

- After this Step we should be able to connect to the switch by entering the default IP Adress into the Browser
  Once we see the switches Login page we enter the default credentials and Login. After that we probably need
  to setup new credentials directly. !Save these in any case !


- Next step is to check the current firmware version. If it suggests that there
   is a newer version we search for the firmware update download from
   the vendors page. Once downloaded we unzip the package and upload
    the binary to the switch and do the upgrade.

    In most cases there are 2-slots that hold the old firmware version and
    one that holds the newer version we want to apply. If after the reboot
    something went wrong, we can easily switch back to the old version
    the reboot probably also does that automatic, in case of a problem.
  
- We need to apply a defined IPv6 (external) and also IPv4 (internal) 
   
What IPv4 and what Subnetmask do we give the Switch ?

   This is tocco specific... and is defined in Docs DOCS
   Under "IP Ranges" we can see one IPv4 and one IPv6 entrance:
         - IPv4: 10.27.5.0/24

- So we set the IPv6 and the IPv4 to the one given by the docs. This means also
   that we need to change the IP-Address of our local machine again if we would
   still want to connect to it via Ethernet cable and one of its ports.
   This and the next step may vary, depending on the specific switch


- Further down in the Docs DOCS there is a entrance about
   VLAN setup. VLAN setup in Tocco means, on internal VLAN and one for the Guest Network.

   This enables us to use one set of Hardware but create two separate networks in a way. This works
    because the switch gives the network packages different Tags depending on what VLAN they belong to.
    Like that different access levels can be created.

    general VLAN is always on Port 1 (defautl) and the Guest one on 481 ( see docs for details)
    Using VLANs means, all components in the network need to be compatible/manageable.
