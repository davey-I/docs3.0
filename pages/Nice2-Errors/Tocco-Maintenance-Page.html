Tocco-Maintenance-Page

Display-Maintenance-Page

In between Release-Changes a Maintenance-Page gets displayed automatically.
Sometimes it might be necessary to do that also, because a page is not ava. or smth.
Therefore there is a script in the Tocco-gitlab

Tocco Docs: Maintenance-Page

The script is hosted as a gitlab repository and available under
HERE

- First we have to clone the repo

- Inside the cloned repo, there is the mntnc script

- We then have to login to the project with oc login



# Execute the script, without any option 
./mntnc start

# Execute the script, with options
./mntnc start [-c tocco] [-t "24.12.2020 6:00"] [-r 1]




Where :

-c/--customer: branch picked of repository where the customer theme is defined.
If no argument is passed or the branch does not exist the default branch is taken.

-t/--till: date/time until the application is unavailable If the argument is present, the message is Please
try again on [date/time]. Else the more generic message Please try again later. is shown.

-r/--replicas: number of replicas deployed (default: 1)

- Disable/enable bypass as needed via https://master.tocco.ch/_maintenance/bypass/

- Disable maintenance page  ./mntnc stop
