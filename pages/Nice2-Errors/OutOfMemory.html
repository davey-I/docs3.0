Auto_Gen_Table_Name_To_Big

Error Messages

error | Autogenerated table name too long for relation model 'Fdp_membership_registration_widget_config--[>relMembership_question]--[
--Membership_question'. Please consider adding it manually.
+
| warn | Adding to the inverse side of a many to many relation (relOutput_template_field) might be slow
 ... ending in a OOM

What the Error Means

     
  •  Die Fehlermeldung Autogenerated table name too long... wird bei jedem (Neu)start der Applikation einmal geloggt, analog der
     Meldungen Adding to the inverse side of a many to many relation (relOutput_template_field) might be slow. . Das führt nicht zum
     Problem mit dem Widget, sondern tritt auf, weil das Widget wahrscheinlich einen OOM (oder sonst ein Problem) verursacht und danach
     das Tocco neu startet
  • Köntte in dem Sinn gefixt werden, wennn die 2-Tablenames kürzer wären

Why the Error Happens

     
  • Database systems (like PostgreSQL) limit table names to 63 characters (or 64 in MySQL).
    •  
  • Es wird glaub sowas wie Table_name_relOther_table_name en "Link" (DB-techinsch, nöd URL) baut, und nachher häsch amel so näme:
     'Extended_teaching_profession_qualification_address--[>relTeaching_profession_qualification]--[qualification_address]--Teaching_profession_qualification'
