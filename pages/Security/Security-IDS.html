An Intrusion Detection System is more of an alerting system that lets an organization know if anomalous or malicious activity
 is detected. An Intrusion Prevention System takes this detection a step forward and shuts down the network before access can be
 gained or to prevent further movement in a network.


IDS

  • IDS use two types of detection:    
             
    • Signature-based detection: alert based on pre-existing signatures that refer to a type of attack or malicious behavior.

      •        
    • Anomaly-based detection: monitoring malicious or suspicious patterns of behavior. This allows them to detect new
      kinds of threats, which is nearly impossible for signature-based systems.    
               
      • For example, if you have set normal working hours for employees, an anomaly-based IDS may flag a login occurring over the
        weekend. The system may also alert you based on the amount of traffic connecting to your network, or new devices
        being added without the right authorization.
        •    
           
      •    
    • IDS types vary based on where they’re monitoring threats and how they’re detecting them.
       
    • 1. Network Intrusion Detection Systems (NIDS)    
             
      • A network intrusion detection system monitors traffic through various sensors — placed either via hardware or software — on the network itself.
        •      
      • It observes all traffic going through devices across multiple sensor points.
        •    
       
      •  
    • 2. Host Intrusion Detection Systems (HIDS)    
             
      • A HIDS is installed directly on endpoints (e.g., servers, workstations) to monitor traffic and system activity.
        •      
      • Offers more control and flexibility to network administrators.
        •      
      • However, it can become burdensome at scale, since each device must be individually monitored and maintained.
        •    
       
      •  
    • 3. Protocol-Based Intrusion Detection Systems (PIDS)    
             
      • Placed at the front of a server to monitor specific communication protocols.
        •      
      • Helps secure user interaction, especially in web-based environments.
        •    
       
      •  
    • 4. Application Protocol-Based Intrusion Detection Systems (APIDS)    
             
      • Similar to PIDS, but designed to monitor traffic across groups of servers using specific application-level protocols.
        •      
      • Helps network administrators segment, classify, and monitor application-specific activity.
        •    
       
      •  
    • 5. Hybrid Intrusion Detection Systems    
             
      • Combines elements of NIDS, HIDS, PIDS, and APIDS into a unified solution.
        •      
      • Allows broader coverage and centralized visibility using a single management interface.
        •      
      • Common in commercial security suites offering comprehensive intrusion detection.
        •    
       

    IPS

    An IPS has the same functionality as IDS systems in terms of detection but also contains response capabilities.
    An IPS solution has more agency and takes action when a potential attack, malicious behavior, or an
    unauthorized user is detected.

    The specific functions of an IPS depend on the type of solution, but in general, having an IPS in place is helpful
    to automate actions and contain threats without the need for an administrator.

       
    • 1. Network-Based Intrusion Prevention System (NIPS)    
             
      • Monitors and protects an entire network from anomalous or suspicious behavior.
        •      
      • Broad-based system often integrated with other monitoring tools.
        •      
      • Helps provide a comprehensive view of an organization's network activity and threats.
        •    
       
      •  
    • 2. Wireless Intrusion Prevention System (WIPS)    
             
      • Specialized in monitoring and protecting wireless networks.
        •      
      • Similar in function to NIPS but limited to wireless spectrum and Wi-Fi devices.
        •      
      • Useful for securing office Wi-Fi, guest networks, and detecting rogue access points.
        •    
       
      •  
    • 3. Host-Based Intrusion Prevention System (HIPS)    
             
      • Deployed on individual hosts (e.g., servers, endpoints).
        •      
      • Monitors incoming and outgoing traffic from the host to detect and block malicious activity.
        •      
      • Often used to protect critical assets and high-value systems.
        •    
       
      •  
    • 4. Network Behavioral Analysis (NBA)    
             
      • Focuses on detecting anomalous patterns of behavior within the network itself.
        •      
      • Key in identifying DDoS attacks, policy violations, or stealthy malware activity.
        •      
      • Rather than using signatures, it relies on behavior profiling and anomaly detection.
        •
