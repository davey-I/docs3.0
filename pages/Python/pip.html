Problem 1: pip install gives externally-managed-environment error

This happens because newer versions of Debian (12+) and Ubuntu (23.04+) follow PEP 668. This Python Enhancement Proposal prevents users from installing packages system-wide using pip, in order to avoid breaking system-managed Python tools.

System Python is now considered a shared resource between the operating system and the user. If you modify it with pip install, you risk introducing version conflicts or bugs into OS utilities that rely on Python.

Solution: Use a Virtual Environment (Recommended)

This keeps your Python packages isolated from the system Python, avoiding all conflicts.

python3 -m venv ~/.venvs/grafana
source ~/.venvs/grafana/bin/activate
pip install grafana-client

Alternative: Use pipx for CLI Tools

pipx is a Python tool that installs Python applications in isolated environments, but makes their command-line interfaces globally accessible.

sudo apt install pipx
pipx install grafana-client

Alternative: Use apt install

Install the package via apt (if available in apt)

sudo apt install pipx
# xyz would in our exmample be grafana-client

apt install python3-xyz
Possible Problem ❌: apt install python3-grafana-client Fails

The Debian/Ubuntu repositories do not contain a package named python3-grafana-client. That’s why the command fails with:

E: Unable to locate package python3-grafana-client

This is common for many Python packages that are not maintained or packaged by Debian maintainers. In such cases, pip (or pipx) is the only way to install them.

Advanced: Override Debian's Protection (Not Recommended)

You can bypass the protection, but this risks corrupting your system Python environment. Only do this if you understand the consequences and have no better options.

pip install --break-system-packages grafana-client

Conclusion

The externally-managed-environment error is not a bug, but a deliberate protection mechanism introduced by Debian and Ubuntu. It's meant to keep your system stable and safe. The best practice is to use a virtual environment or pipx for any Python development or CLI tools, rather than trying to install them system-wide.
